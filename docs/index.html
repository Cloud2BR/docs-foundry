Open-source documentation workspace

Docs Foundry

A desktop documentation editor with live Markdown preview, sidebar file tree, and cross-platform installers. Built with Electron and a container-first development workflow.

Getting started

What is Docs Foundry?

Docs Foundry is an open-source desktop application for writing, editing, and previewing Markdown documentation. Think of it as a lightweight, local-first docs workspace. Like Obsidian, but purpose-built for documentation projects.

📝

Markdown editor

Write in a clean, distraction-free text editor with monospace font and syntax awareness. Save with Ctrl+S and track unsaved changes automatically.

👁️

Live preview

See your Markdown rendered in real time in a split pane beside the editor. Supports headings, bold, italic, code blocks, lists, links, images, and blockquotes.

📂

Sidebar file tree

Open any folder and browse its entire file tree in a collapsible sidebar. Click files to open them, and navigate deep directory structures up to 8 levels.

🐳

Container-first dev

Contributors never install Node.js locally. Run make setup to build a Docker container and install all dependencies inside it.

📦

Cross-platform binaries

Ship native installers for macOS (DMG), Windows (NSIS), and Linux (AppImage, DEB). Built automatically via GitHub Actions.

🔄

Auto-update

Packaged builds check GitHub Releases for updates via electron-updater. Users get the latest version without manual downloads.

For contributors

Quick start

Everything runs inside Docker. No local Node.js or npm required.

  1. Install Docker Get Docker Desktop or Docker Engine for your OS.
  2. Clone and setup git clone https://github.com/Cloud2BR/docs-foundry.git && cd docs-foundry && make setup
  3. Open a dev shell make container-shell. You're inside the container with all tools ready.
  4. Run quality checks make lint, make format, make test. All run inside the container.
